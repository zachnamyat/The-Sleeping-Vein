# Fonts

The project uses **two pixel fonts** to give the UI a body voice and a lore voice:

| Font | Role | Designer | License | Origin |
|------|------|----------|---------|--------|
| **m5x7** | Default body ‚Äî HUD, hotbar, inventory, buttons, dialogue speaker names, tooltips | Daniel Linssen | CC BY 4.0 | [managore.itch.io/m5x7](https://managore.itch.io/m5x7) |
| **Monogram** | Lore flavor ‚Äî dialogue body, boss intro crawl, opening subtitle, compendium tablet/relic/title tabs, tooltip lore lines, sovereign-name flavor text | datagoblin | CC0 1.0 | [datagoblin.itch.io/monogram](https://datagoblin.itch.io/monogram) |
| **Monogram Extended** | Reserved for diacritics / extended Latin support (loaded on demand) | datagoblin | CC0 1.0 | [datagoblin.itch.io/monogram](https://datagoblin.itch.io/monogram) |

Both fonts ship as `.ttf` files in this directory with Godot import settings tuned for pixel-perfect rendering (`antialiasing=0`, `hinting=0`, `subpixel_positioning=0`, `multichannel_signed_distance_field=false`).

## Render sizes

- **m5x7 at size 16** ‚Äî body default. Designed at multiples of 16; lower sizes blur.
- **Monogram at size 18** ‚Äî lore flavor. Designed at multiples of 9; size 18 reads at a similar visual height to m5x7 size 16.
- **m5x7 at size 32** ‚Äî the AETHERDEEP title in the opening sequence.

`resources/themes/sleeping_vein.tres` binds m5x7 as the project-wide default. Monogram is applied per-node via `theme_override_fonts/font` on the specific lore surfaces ‚Äî see:

- `scenes/ui/dialogue_panel.tscn` ‚Äî Body RichTextLabel
- `scenes/ui/boss_intro_crawl.tscn` ‚Äî Crawl Label
- `scenes/ui/opening_sequence.tscn` ‚Äî Subtitle Label
- `scenes/ui/tooltip.tscn` ‚Äî Lore Label (sovereign-name + relic flavor)
- `scripts/ui/compendium_panel.gd` ‚Äî runtime-built Tablet / Relic / Title list entries

## Swap procedure (if you want to audition a different font)

1. Drop the `.ttf` into `assets/fonts/<your_font>.ttf`.
2. Pre-create the `.import` file with `antialiasing=0`, `hinting=0`,
   `subpixel_positioning=0`, `multichannel_signed_distance_field=false`,
   `keep_rounding_remainders=true`, `oversampling=1.0` (or copy
   `m5x7.ttf.import` as a template).
3. Run `godot --headless --path . --import` so Godot materializes the
   `.ctex/.fontdata` cache and assigns a UID.
4. Open `resources/themes/sleeping_vein.tres` and swap the `ExtResource` path
   to the new font (and the UID Godot just assigned).
5. Reload the project. Every UI label / button picks up the change because the
   theme is bound globally via `gui/theme/custom` in `project.godot`.

## Choosing a render size for a new font

Pixel fonts render cleanly only at integer multiples of their design grid:

- m5x7 ‚Üí 16, 32, 48, 64 ‚Ä¶
- Monogram ‚Üí 9, 18, 27, 36 ‚Ä¶
- PixelOperator ‚Üí 16, 32 ‚Ä¶
- m6x11 ‚Üí 11, 22, 33 ‚Ä¶

Pick the smallest multiple that's still readable at 480√ó270 (typically the 2√ó
size ‚Äî m5x7 at 16, Monogram at 18). The `font_size` parameter in Godot's theme
is per-control class, so you can pin individual control classes to a different
size than `default_font_size`.
